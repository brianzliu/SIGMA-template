# SIGMA — Simulator-Interface Grounding Adapter Template

A Claude Code plugin template that wraps `claude` with four grounding
components so a model can author runnable input files for an arbitrary
scientific simulator:

1. **RAG** — ChromaDB-backed retrieval over the simulator's docs, schema, and examples
2. **Linter** — a Stop-hook + agent MCP tool that validates the agent's output against the simulator's grammar
3. **Memory primer** — a system-prompt prefix that orients the model on file layout, top-level structure, and search workflow
4. **Self-refinement** — the same Stop-hook that re-enters the agent with the linter's feedback when validation fails at end-of-turn

This template was extracted from the original GEOS-targeted research repo at
[matt-seb-ho/repo3](https://github.com/matt-seb-ho/repo3). The first
adaptation was [brianzliu/repo3_openfoam](https://github.com/brianzliu/repo3_openfoam).
Both ship under [`examples/`](examples/) as worked configurations.

## Quick start

```bash
git clone https://github.com/brianzliu/SIGMA-template my-sim-plugin
cd my-sim-plugin

# 1. Pick a starting point
cp examples/geos/simulator.yaml configs/simulator.yaml     # or openfoam/
# (or edit configs/simulator.yaml from scratch)

# 2. Apply the config — renames files, substitutes placeholders, writes plugin.json
python scripts/init_template.py

# 3. Implement the simulator-specific bits
$EDITOR plugin/adapters/validator.py       # what counts as a valid output
$EDITOR plugin/adapters/rag_indexer.py     # how to chunk + index docs/examples

# 4. Build the RAG ChromaDB indices
python scripts/build_chromadb.py

# 5. Test the plugin against Claude Code
cd plugin && claude --plugin-dir .
```

## What you fill in vs. what's templated

| You fill in | Templated automatically by `init_template.py` |
|---|---|
| `configs/simulator.yaml` | Plugin name, MCP server name, env var prefix in `plugin/.claude-plugin/plugin.json` |
| `plugin/adapters/validator.py` (3 functions) | All `GEOS_*` env var names in primers, skill, hooks |
| `plugin/adapters/rag_indexer.py` (3 functions) | Skill directory rename: `plugin/skills/sim-rag/` → `plugin/skills/<slug>-rag/` |
| The body of `plugin/PRIMER_*.md` (your simulator's file layout, top-level structure, examples) | RAG MCP script rename: `plugin/scripts/rag_mcp.py` → `plugin/scripts/<slug>_rag_mcp.py` |
| | Primer rename: `plugin/PRIMER_*.md` → `plugin/<SLUG>_PRIMER_*.md` |

## Repository layout

```
configs/
  simulator.yaml          # the one config that drives everything
examples/
  geos/simulator.yaml     # original (multiphysics XML; xmllint --schema)
  openfoam/simulator.yaml # first adaptation (case dirs; structural check)
plugin/
  .claude-plugin/
    plugin.json           # generated by init_template.py
  PRIMER_absolute_min.md  # system-prompt prefix variants — fill in your simulator
  PRIMER_minimal.md
  PRIMER_minimal_vanilla.md
  adapters/
    validator.py          # simulator-specific output validation
    rag_indexer.py        # simulator-specific doc chunking + indexing
  hooks/
    hooks.json
    verify_outputs.py     # Stop-hook self-refinement (calls validator)
    verify_post_write.py  # PostToolUse hook (calls validator on each write)
  scripts/
    rag_mcp.py            # ChromaDB MCP server (renamed to <slug>_rag_mcp.py)
    validator_mcp.py      # agent-callable validator MCP tool
    memory_mcp.py         # cheatsheet/memory MCP server
    supervisor_mcp.py     # optional: subagent supervisor
  skills/
    sim-rag/SKILL.md      # RAG skill (renamed + edited by init_template.py)
    plugin-maintainer/    # generic skill, unchanged
src/runner/               # the experiment harness — generally simulator-agnostic
scripts/
  init_template.py        # the customization script
  build_chromadb.py       # populate the three RAG collections (you write this)
  run_and_eval.py         # batch runner
  run_experiment.py       # single-task runner
```

## Adapting to a new simulator

See [`docs/ADAPTATION_GUIDE.md`](docs/ADAPTATION_GUIDE.md). The short version:

1. Fork or copy this repo.
2. Write a new `configs/simulator.yaml` (use `examples/openfoam/simulator.yaml` as a closer-to-yours starting point).
3. Run `python scripts/init_template.py --dry-run` and read the planned changes.
4. Run it for real.
5. Implement the three functions in `plugin/adapters/validator.py` against your simulator's parser/lint tool.
6. Implement the three functions in `plugin/adapters/rag_indexer.py` against your simulator's docs/schema/examples.
7. Author the three `PRIMER_*.md` variants from your simulator's docs.

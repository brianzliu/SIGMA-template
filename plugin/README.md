# sigma-{{SLUG}}-plugin

Claude Code plugin for {{SIMULATOR_NAME}} authoring, generated from
the SIGMA template.

## Structure

```text
.
├── .claude-plugin/
│   └── plugin.json           # auto-generated by scripts/init_template.py
├── adapters/
│   ├── validator.py          # simulator-specific output validation
│   └── rag_indexer.py        # simulator-specific doc indexing
├── hooks/
│   ├── hooks.json
│   ├── verify_outputs.py     # Stop hook: re-enters agent on validation failure
│   └── verify_post_write.py  # PostToolUse hook: fast per-write parse check
├── scripts/
│   ├── {{SLUG}}_rag_mcp.py   # ChromaDB MCP server (3 search_* tools)
│   ├── validator_mcp.py      # agent-callable {{LINT_MCP_TOOL}} tool
│   ├── memory_mcp.py         # cheatsheet/memory MCP server
│   └── supervisor_mcp.py     # optional: subagent supervisor
├── skills/
│   ├── {{SLUG}}-rag/SKILL.md # how the model should use the search_* tools
│   └── plugin-maintainer/    # generic skill, unchanged from upstream
├── {{SLUG_UPPER}}_PRIMER_absolute_min.md   # system-prompt prefix variants
├── {{SLUG_UPPER}}_PRIMER_minimal.md
└── {{SLUG_UPPER}}_PRIMER_minimal_vanilla.md
```

## Local testing

From the repo root:

```bash
claude --plugin-dir plugin
```

Inside Claude Code:

```text
/reload-plugins
/sigma-{{SLUG}}-plugin:plugin-maintainer
```

## Environment variables

| Var | Default | Purpose |
|---|---|---|
| `{{ENV_PREFIX}}_HOOK_DISABLE` | `0` | Kill switch — disables both Stop and PostToolUse hooks. |
| `{{ENV_PREFIX}}_HOOK_INPUTS_DIR` | `$CLAUDE_PROJECT_DIR/inputs` | Where the agent writes outputs. |
| `{{ENV_PREFIX}}_HOOK_MAX_RETRIES` | `2` | How many times the Stop hook blocks before giving up. |
| `{{ENV_PREFIX}}_HOOK_SCHEMA_PATH` | `{{SCHEMA_PATH}}` | Schema/spec passed to `validator.validate_file`. |
| `{{ENV_PREFIX}}_VECTOR_DB_DIR` | `{{VECTOR_DB_DIR}}` | ChromaDB location for RAG. |

Use `${CLAUDE_PLUGIN_ROOT}` for files shipped with the plugin and
`${CLAUDE_PLUGIN_DATA}` for persistent generated state.
